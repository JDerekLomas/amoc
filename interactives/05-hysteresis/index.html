AMOC hysteresis loop in the spatial JAX model

The two-box Stommel interactive shows the AMOC bifurcation in two ODEs. The same physics, in a 256×128 spatially-resolved coupled model with real continents, takes ~30 minutes to trace out. Here's the result of one quasi-static sweep.

No data yet — run python amoc-jax/scripts/hysteresis.py to generate curve.json in this directory. The Python sweep takes ~30 minutes (32 segments × 4000 steps × 5e-5 dt) and produces a (F, AMOC) trajectory of the up-leg and down-leg.

The procedure

1. Spin the simulator up to its baseline steady state with F = 0.
2. Ramp up: increase freshwater_forcing from 0 → 5 in 16 segments. At each segment, run the model for 4000 timesteps so it approximately tracks quasi-equilibrium for the new F. Record (F, AMOC, N. Atl SST).
3. Ramp down: decrease F from 5 → 0 the same way.
4. Plot AMOC strength against F: an up-leg path and a down-leg path. If the salt-advection feedback is producing bistability, the two legs do not coincide — that's the hysteresis loop.

Why this matters

A reversible loop means the model is just a damped linear system. A non-reversible (open) loop means there are two stable AMOC states — the thermal mode (active overturning, ψ > 0) and the salt mode (collapsed, ψ ≈ 0 or reversed) — separated by a saddle-node bifurcation.

This is exactly the picture Stommel (1961) predicted, Rahmstorf (1996) demonstrated experimentally in EMICs, and recent work (van Westen, Kliphuis & Dijkstra 2024 in Sci Adv, Kuhlbrodt & Dijkstra 2025 in ESD) has shown is also present in CMIP-class GCMs. Reproducing it in our toy model — at < 1 minute of integration per F-value — confirms the JAX core captures the physics, not just the visuals.
